Module Scan · how it works

Finding the camera hidden in a pair of glasses.

A tool that looks at a photo of eyeglasses and decides: are these Ray-Ban Meta smart glasses — the ones with a tiny camera in the frame corner — or just normal glasses?

CORNER · bezel + glint + dark core

The core idea

Two kinds of intelligence, working together

There are two ways to make a computer recognise something. This project uses both — each exactly where it's strongest.

▪ Classical computer vision · ~95%

Rules we wrote down

Step-by-step instructions, like a recipe: find the edges, find the circles, measure the darkness. Used to locate the frame, the two corners, and any circular shapes. Great at measuring things.

▪ Machine learning · one step

A pattern it learned itself

We show it example photos and it works out the difference. Used for one call only: is this corner a real camera, or just a rounded frame corner?

Why both? A camera lens and a rounded frame corner are both “a dark circle in the corner.” No rule can separate them — the difference is about shape and texture, not darkness. So for that one judgment, the computer learns from examples instead of following a rule.

The learned step, up close

How the machine learning actually works

Machine learning means learning a pattern from labelled examples instead of being told the rule. This model does it in three stages.

A · turn picture into numbers

Features (HOG)

The corner is shrunk to a 32×32 patch and turned into 144 numbers that describe which way the edges point. A round bezel produces very different numbers than a bare L-shaped corner.

Like describing a face by the angles of its features, not the raw pixels.
B · the model

Logistic regression

The simplest learning machine. It learns a weight for each of the 144 numbers, adds them up, and squashes the total into a probability between 0 and 1. The whole model is a few hundred numbers in one small file.

Deliberately simple — a bigger model would just memorise 19 photos.
C · training

Gradient descent

Weights start at zero (pure guessing). It sees each example, measures how wrong it was, and nudges the weights a little less wrong — repeated 2,000 times.

This same core idea powers all machine learning — here, at a tiny scale.

What I actually did

How I trained the model

01

Collected examples. Photos sorted into two folders: ray_ban_frame/ (smart glasses — the “yes” examples, or positives) and normal_frame/ (normal glasses — the negatives).

02

Labelled them — just by which folder they live in. The folder is the answer key.

03

Made more out of less. With only 19 images, each one is copied with small changes — brightness, shift, rotation — so the model sees variety instead of memorising exact pixels. This is called augmentation.

04

Ran train_camera_clf.py — it extracts the numbers, runs the gradient descent, and saves the learned weights to models/camera_clf.npz.

▪ Known weak spots (worth volunteering)

Strongly angled / side-on photos, and “cat-eye” normal frames whose pointed corner mimics a camera module. More training photos would push the accuracy higher.
